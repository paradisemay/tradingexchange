# ClickHouse Contracts

## Go to ClickHouse

Transport: ClickHouse native TCP.

Default local endpoint:

```text
127.0.0.1:9000
```

Target table:

```text
trading.quotes_raw
```

Insert columns:

| Column | Type | Required | Description |
| --- | --- | --- | --- |
| `symbol` | `LowCardinality(String)` | yes | Normalized ticker from C driver |
| `event_time` | `DateTime64(3, 'UTC')` | yes | Quote event time from Go reader |
| `price` | `Float64` | yes | Quote price |
| `ingested_at` | `DateTime64(3, 'UTC')` | yes | Insert time generated by Go |

Batching:

- maximum batch size: `CLICKHOUSE_BATCH_SIZE`, default `10000`;
- flush interval: `CLICKHOUSE_FLUSH_INTERVAL_MS`, default `200`;
- insert timeout: `CLICKHOUSE_INSERT_TIMEOUT_MS`, default `5000`;
- retry backoff: `CLICKHOUSE_RETRY_INITIAL_MS` to `CLICKHOUSE_RETRY_MAX_MS`.

## ClickHouse to Ktor

Ktor is the only backend reader. Mobile clients must not query ClickHouse directly.

Raw points query source:

```text
trading.quotes_raw
```

Candles query source:

```text
trading.quotes_ohlc_1m_read
```

Required API shapes are documented in `clickhouse/openapi/ktor-quotes-history.yaml`.
